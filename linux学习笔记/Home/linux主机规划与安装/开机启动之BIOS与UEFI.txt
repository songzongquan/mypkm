====== 启动过程：BIOS与UEFI区别 ======
Created 星期六 21 四月 2018

BIOS和UEFI都属于固件。BIOS是老技术，UEFI是新的技术
CMOS是保存硬件配置参数的一个存储器，而BIOS是主机加电后自动调用运行的第一个程序。

====== BIOS启动过程 ======
简单的说,整个开机流程到操作系统之前的动作应该是这样的:

1. BIOS:开机主动执行的固件,会认识第一个可开机的设备;
2. MBR:第一个可开机设备的第一个扇区内的主要开机记录区块,内含开机管理程序;
3. 开机管理程序( boot loader) :一支可读取核心文件来执行的软件;
4. 核心文件 :开始操作系统的功能...

BIOS 可以从GPT分区中的LBA0这一个MBR兼容 区块加载开机管理程序，如果你的开机管理程序支持GPT，那么它也能正常引导。

===== 开机管理程序 =====
又称为boot loader，它的作用如下：
1. 提供菜单 :使用者可以选择不同的开机项目,这也是多重开机的重要功能!
2. 载入核心文件 :直接指向可开机的程序区段来开始操作系统;
3. 转交其他 loader:将开机管理功能转交给其他loader 负责。

实际上每一个分区都自已的启动扇区，上面都可以安装自己的开机管理程序，所以一个硬盘上可以安装多个操作系统。
上面第3步所说就是这种情况。

====== UEFI的启动过程 ======

UEFI(Unified Extensible Firmware Interface) 
突破了BIOS的限制 ，比如它直接支持GPT分区。它用以取代BIOS，它用c语言开发，BIOS用汇编开发，它更好维护，功能更强大。界面豪华。
启动过程和BIOS的步骤差不多，主要区别是它突破了一些空间限制， 开机管理程序，可以达到256-2t的空间内。
UEFI甚至 开发的可以象一个小操作系统一样。
